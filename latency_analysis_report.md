# AI 음성 차팅 파이프라인 레이턴시(속도) 분석 보고서
이 파일은 음성 분석 서비스의 각 처리 단계별 지연 요인을 상세 분석하고 이를 대폭 단축하기 위한 구체적인 최적화 방안을 제시합니다.

---

## 1️⃣ 단계별 레이턴시 소요 시간 및 병목 요인 분석

전체 파이프라인은 크게 **5개 단계**로 작동하며, 각 단계별 평균 소요 시간 및 원인은 다음과 같습니다 (30분 오디오 청크 기준).

```mermaid
gantt
    title 30분 오디오 분석 레이턴시 타임라인 (평균 90초)
    dateFormat  SS
    axisFormat %S초
    section 파일 전송
    1. 브라우저 업로드 (15MB)      :active, a1, 00, 15s
    section STT 엔진
    2. Soniox STT + 다이아리제이션:active, a2, 15, 45s
    section LLM 분석
    3. OpenAI GPT 텍스트 정제    :active, a3, 60, 10s
    4. OpenAI SOAP 차트 생성     :active, a4, 70, 12s
    5. OpenAI 가이드 생성        :active, a5, 82, 08s
```

### 1-1. 브라우저 오디오 업로드 (지연: 5 ~ 20초)
- **원인**: 30분 분량의 원본 오디오가 대용량(WAV 기준 약 150MB~300MB)일 경우, 사용자의 인터넷 업로드 속도에 따라 병목이 크게 발생합니다.
- **Opus 압축 시**: 약 15MB 이하로 줄어들어 3~5초 이내에 완료되나 여전히 초기 지연의 큰 축을 차지합니다.

### 1-2. Soniox STT & 화자 구분 (지연: 30 ~ 50초)
- **원인**: 음성을 텍스트로 바꾸고 화자(강사/고객)를 매핑하는 작업은 오디오 크기에 비례해 처리 시간이 늘어납니다. Soniox의 비동기 전사 기능은 일반적으로 오디오 길이의 약 **2%~3%** 시간에 전사를 끝내므로, 30분 분량은 기계 학습 처리 시간에만 최소 30초 이상이 소요됩니다.

### 1-3. OpenAI GPT API 호출 (지연: 25 ~ 35초)
- **원인**: 현재 파이프라인은 GPT API를 **3번 연속(순차적)**으로 호출하고 있습니다:
  1. `정제(Refine)` 대기 및 응답 수신 (약 10초)
  2. `SOAP 차트(JSON)` 생성 대기 및 응답 수신 (약 12초)
  3. `환자 가이드` 생성 대기 및 응답 수신 (약 8초)
  - 이 순차적인 API 왕복 호출 시간이 누적되어 약 30초의 큰 지연이 고스란히 추가됩니다.

---

## 2️⃣ 속도 단축을 위한 구체적인 최적화 솔루션

### 2-1. GPT API 병렬 처리 (즉시 20초 단축 가능)
- **개선안**: 정제가 끝난 텍스트를 기반으로 **SOAP 차트 작성**과 **환자용 가이드 작성**을 `Promise.all`을 이용해 **동시에(병렬로) OpenAI에 요청**합니다.
- **효과**: 기존에 30초 이상 걸리던 GPT 분석 지연 시간이 **최대 15초 내외로 단축(약 50% 절감)**됩니다.

### 2-2. 클라이언트단 오디오 압축 포맷 강제 (업로드 지연 최소화)
- **개선안**: 브라우저에서 녹음이나 업로드 시 원본 WAV가 아닌 고압축 무손실 코덱인 **Opus (webm) 포맷**을 강제하고 샘플링 레이트를 16kHz 모노로 제한하여 용량을 원본의 10분의 1 수준으로 다이어트시킵니다.
- **효과**: 네트워크 대역폭 병목이 사라져 대용량 파일도 3초 이내에 클라우드 스토리지에 업로드됩니다.

### 2-3. Soniox 비동기 전사 모델 세팅 최적화
- **개선안**: 실시간성이 매우 중요한 빠른 전사의 경우, Soniox의 비동기 모델 옵션 중 더 최적화된 저지연 모델(`stt-async-v5` 또는 그 이상)로 업그레이드하고 필요 없는 음성 평가 파라미터를 비활성화합니다.

---

## 3️⃣ 최종 결론 및 요약
- 현재 음성 분석이 1분~1.5분 정도 소요되는 주원인은 **① STT 엔진 고유의 음성 연산 대기 시간**과 **② GPT API를 3회 연속 순차 호출하는 병목** 때문입니다.
- 이 중 **GPT 병렬 호출 기법**을 이식하는 것만으로도 대기 시간을 즉시 20초 이상 극적으로 줄여 실시간 반응 속도를 크게 올릴 수 있습니다.
